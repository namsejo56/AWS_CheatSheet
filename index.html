🖥️ 1. COMPUTE SERVICES

EC2
Amazon EC2 (Elastic Compute Cloud)

Instance Types & Use Cases

Family Type Use Case Example
General Purpose t3/t4g, m6i, m7i, m7g, m7i-flex Balanced compute & memory for web/app tiers, microservices m7i-flex.large (2 vCPU, 8 GiB RAM)
Compute Optimized c6i, c7g, c7gn, Hpc7g High-performance & network intensive workloads c7gn.16xlarge (64 vCPU, 128 GiB RAM, 200 Gbps)
Memory Optimized r6i, r7iz, x2idn/x2iedn, u-24tb1 In-memory databases, large SAP HANA footprints r7iz.4xlarge (16 vCPU, 128 GiB RAM)
Storage Optimized i4i, im4gn, is4gen, d3 Low-latency transactional systems, log analytics i4i.4xlarge (16 vCPU, 128 GiB RAM, NVMe SSD)
Accelerated Computing p5, g6, trn1, inf2 Generative AI, ML training/inference, 3D rendering p5.48xlarge (8 NVIDIA H100 GPUs, 768 GiB RAM)

Pricing Models

  • On-Demand: Pay per second, no commitment. Most expensive but flexible.
  • Reserved Instances (RI): 1-3 year commitment, up to 75% savings
    • Standard RI: Highest discount, can't change instance type
    • Convertible RI: Can change instance family, lower discount
    • Scheduled RI: For predictable workloads at specific times
  • Spot Instances: Up to 90% discount, can be terminated by AWS
  • Dedicated Hosts: Physical server dedicated to you, compliance requirements
  • Dedicated Instances: Your instances run on dedicated hardware
  • Savings Plans: Commitment to compute usage ($/hour) for 1-3 years

Exam Tips:
  • T3 instances are burstable - good for variable workloads
  • Spot instances perfect for fault-tolerant workloads
  • Reserved Instances best for steady-state workloads
  • Use placement groups for low latency (Cluster) or high availability (Spread)
2024-2025 Updates:
  • EC2 M7i-flex and M7i deliver up to 15% better price/performance than M6i and support automatic network bandwidth scaling to 50 Gbps.
  • Preview R8g instances powered by Graviton4 boost memory bandwidth by 50% and add DDR5 ECC protection for SAP and in-memory analytics workloads.
  • EC2 P5 (NVIDIA H100) and G6 (L40S) target next-generation AI workloads with Capacity Blocks reservations for short-lived training runs.
  • C7gn and Hpc7g provide ultra-low latency networking with Elastic Fabric Adapter (EFA) for HPC and real-time packet processing.

Event Sources & Triggers

  • Synchronous Invocation: API Gateway, ALB, CLI, SDK
  • Asynchronous Invocation: S3, SNS, EventBridge, SES
  • Poll-based Invocation: Kinesis, DynamoDB Streams, SQS

Pricing Model

Lambda Pricing Components:
  • Requests: $0.20 per 1M requests
  • Duration: $0.0000166667 per GB-second
  • Free Tier: 1M requests + 400,000 GB-seconds/month

Example: 1M requests, 512MB, 3s duration = $0.20 + (1M × 0.5GB × 3s × $0.0000166667) = $25.20/month

Lambda Best Practices:
  • Keep functions small and focused (single responsibility)
  • Minimize cold starts by keeping functions warm
  • Use environment variables for configuration
  • Implement proper error handling and retry logic
  • Use Lambda layers for shared code/dependencies
Latest Enhancements:
  • Lambda SnapStart (GA) cuts Java cold starts by up to 10x by restoring from pre-initialized snapshots.
  • Lambda Function URLs with IAM or JWT authentication provide dedicated HTTPS endpoints without API Gateway.
  • Response Streaming (2023) enables streaming payloads back to clients—ideal for generative AI applications.
  • Configurable ephemeral storage up to 10 GiB supports short-lived data processing and ML build tasks.

Security Features

  • Encryption at Rest:
    • SSE-S3: AWS managed keys (AES-256)
    • SSE-KMS: AWS KMS managed keys
    • SSE-C: Customer provided keys
    • Client-side encryption
  • Encryption in Transit: SSL/TLS (HTTPS)
  • Access Control:
    • IAM policies
    • Bucket policies (JSON-based)
    • Access Control Lists (ACLs)
    • Pre-signed URLs (temporary access)
  • Other Security Features:
    • MFA Delete for versioned objects
    • Cross-Region Replication (CRR)
    • Same-Region Replication (SRR)
    • Object Lock (WORM compliance)

Performance Features

  • Request Rate: 3,500 PUT/COPY/POST/DELETE and 5,500 GET/HEAD requests per second per prefix
  • Transfer Acceleration: Uses CloudFront edge locations for faster uploads
  • Multipart Upload: For objects >100MB, required for objects >5GB
  • Byte-range Fetches: Download specific byte ranges from objects
S3 Exam Tips:
  • S3 is eventually consistent for overwrite PUTS and DELETES
  • Use lifecycle policies to automatically transition objects between storage classes
  • Cross-Region Replication requires versioning enabled
  • S3 Transfer Acceleration uses CloudFront edge locations
  • S3 Select allows querying data without downloading entire object

🌐 3. NETWORKING & CONTENT DELIVERY

VPC
Amazon VPC (Virtual Private Cloud)

VPC Core Components

Internet Gateway (IGW) ↓ VPC (10.0.0.0/16) ├── Public Subnet (10.0.1.0/24) → Route Table → IGW │ └── EC2 with Public IP └── Private Subnet (10.0.2.0/24) → Route Table → NAT Gateway └── EC2 with Private IP only
  • Subnets:
    • Public: Has route to Internet Gateway
    • Private: No direct route to internet
    • Each subnet resides in single AZ
  • Route Tables: Control traffic routing within VPC
  • Internet Gateway (IGW): Horizontally scaled, redundant connection to internet
  • NAT Gateway: Managed service for outbound internet access from private subnets
  • NAT Instance: EC2 instance acting as NAT (legacy, not recommended)

VPC Security

Feature Security Groups Network ACLs
Level Instance level Subnet level
Rules Allow rules only Allow and Deny rules
State Stateful Stateless
Rule Evaluation All rules evaluated Rules processed in order
Default Denies all inbound, allows all outbound Allows all inbound and outbound

VPC Connectivity Options

  • VPC Peering:
    • Network connection between two VPCs
    • Can be in same or different regions
    • No transitive peering
    • CIDR blocks cannot overlap
  • Transit Gateway:
    • Central hub to connect multiple VPCs and on-premises
    • Supports transitive routing
    • Cross-region peering
    • Route table propagation
  • VPN Gateway:
    • Site-to-Site VPN connections
    • Customer Gateway on-premises
    • BGP or static routing
    • Up to 1.25 Gbps per tunnel
  • AWS Direct Connect:
    • Dedicated network connection to AWS
    • Lower bandwidth costs
    • More consistent network performance
    • Private connectivity to VPC

🔒 4. SECURITY & IDENTITY

IAM
AWS IAM (Identity and Access Management)

IAM Components

  • Users: Individual people or services
    • Long-term credentials (access keys)
    • Console password
    • MFA devices
  • Groups: Collections of users with same permissions
  • Roles: Temporary credentials assumed by AWS services/users
    • Cross-account access
    • Federation (SAML, OpenID Connect)
    • Service roles (EC2, Lambda, etc.)
  • Policies: JSON documents defining permissions
    • Identity-based policies
    • Resource-based policies
    • Permission boundaries

Policy Example

{ "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": [ "s3:GetObject", "s3:PutObject" ], "Resource": "arn:aws:s3:::my-bucket/*", "Condition": { "StringEquals": { "s3:x-amz-server-side-encryption": "AES256" } } } ] }

IAM Best Practices

  • Principle of Least Privilege: Grant minimum permissions needed
  • Enable MFA: Especially for privileged users
  • Use Roles: Instead of long-term access keys
  • Regular Auditing: Review permissions regularly
  • Strong Password Policy: Enforce complex passwords
  • Access Keys Rotation: Rotate access keys regularly
IAM Exam Tips:
  • IAM is global service (not region-specific)
  • Root account has full permissions - don't use for daily tasks
  • Always use roles for EC2 instances instead of embedding credentials
  • Policy evaluation logic: Explicit Deny > Explicit Allow > Default Deny
KMS
AWS KMS (Key Management Service)

Key Types

  • AWS Managed Keys:
    • Created and managed by AWS
    • Free to use
    • Automatic rotation every year
    • Example: aws/s3, aws/ebs, aws/rds
  • Customer Managed Keys:
    • Created and managed by customer
    • $1/month per key
    • Optional automatic rotation
    • Full control over key policies
  • AWS Owned Keys:
    • Used by AWS services internally
    • Not visible to customers
    • No additional charges

KMS Operations

Customer Master Key (CMK) in KMS ↓ (encrypts) Data Encryption Key (DEK) ↓ (encrypts) Your Data (>4KB)

🗄️ 5. DATABASES

RDS
Amazon RDS (Relational Database Service)

Supported Database Engines

Engine Version Max Storage Max IOPS Special Features
MySQL 5.7, 8.0 64 TiB 80,000 Read replicas, automated backups
PostgreSQL 11, 12, 13, 14, 15 64 TiB 80,000 Advanced features, JSON support

SQL Server 2017, 2019, 2022 16 TiB 80,000 Windows authentication

RDS Features

  • Multi-AZ Deployment:
    • Synchronous replication to standby
    • Automatic failover
    • High availability (99.95% SLA)
    • No performance benefit for reads
  • Read Replicas:
    • Asynchronous replication
    • Scale read workloads
    • Cross-region support
    • Can be promoted to master
    • Up to 15 read replicas for Aurora, 5 for others
  • Automated Backups:
    • Point-in-time recovery (1-35 days)
    • Daily snapshots
    • Transaction log backups every 5 minutes
  • Security:
    • Encryption at rest (KMS)
    • Encryption in transit (SSL/TLS)
    • IAM database authentication
    • VPC security groups
RDS Exam Tips:
  • Multi-AZ is for high availability, Read Replicas for scalability
  • RDS handles patching, backups, and maintenance
  • Cannot SSH into RDS instances
  • Use Parameter Groups to configure database settings
DDB
Amazon DynamoDB

DynamoDB Core Concepts

  • Tables: Collection of data
  • Items: Individual records (max 400KB)
  • Attributes: Data fields in items
  • Primary Key:
    • Partition Key (Hash): Single attribute
    • Composite Key: Partition Key + Sort Key
  • Secondary Indexes:
    • Global Secondary Index (GSI): Different partition key
    • Local Secondary Index (LSI): Same partition key, different sort key

Capacity Modes

Mode Read Pricing Write Pricing Use Case
On-Demand $0.25 per million RRU $1.25 per million WRU Unpredictable workloads
Provisioned $0.09 per RCU-hour $0.47 per WCU-hour Predictable workloads

DynamoDB Features

  • DynamoDB Streams: Capture data modification events
  • Global Tables: Multi-region, multi-master replication
  • Point-in-Time Recovery: Restore to any second in last 35 days
  • DynamoDB Accelerator (DAX): In-memory cache (microsecond latency)
  • Transactions: ACID transactions across multiple items

Consistency Models

  • Eventually Consistent Reads: Default, may return stale data
  • Strongly Consistent Reads: Always returns most recent data
  • Transactional Reads: ACID compliance across multiple items

Specialized Database Services

  • Aurora Serverless:

  • Aurora Global Database:

    • Read replicas in multiple regions

SQS Features

  • Message Attributes:

  • Long Polling:

    • More cost-effective than short polling
SQS Exam Tips:
